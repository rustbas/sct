# Changelog

## 0.2.2

- **Breaking:** id задачи = hash(файл + строка), не текст; после обновления — `sct sync --full`
- Исправлено: два TODO с одинаковым текстом больше не схлопываются в одну запись
- Исправлено: `theme.tcss` включается в пакет (pip / pipx)
- `sct list` показывает id; `--no-id` для компактного вывода
- `done` / `reopen` / `show`: короткий префикс hex-id (мин. 4 символа), как в git

## 0.2.1

- TUI: vim-подобные клавиши (`j`/`k`, `g`/`G`, `Ctrl+d`/`Ctrl+u`)
- TUI: тёмная минималистичная тема, без стандартного Header/Footer

## 0.2.0

- Инкрементальный `sync` (по умолчанию) и `sync --full`
- Проверка строки перед `done` / `reopen` (`line_hash`, текст, маркер)
- Коды выхода для скриптов; `sct doctor`, `sct init`
- Поиск корня проекта вверх по дереву (`.sct/`)
- `requirements.txt` для установки зависимостей
- CI: GitHub Actions + интеграционные тесты
- `list --priority`, `list --json` с полем `version`
- TUI: предупреждение при устаревшем кэше

## 0.1.0

- Первый рабочий прототип: сканер, кэш, CLI, TUI
